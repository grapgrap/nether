# Post

발행 가능한 글의 단위

## 목적

블로그 콘텐츠를 구조화된 형태로 관리하고, 발행 상태를 제어한다.

### 핵심 시나리오

- 작성자가 마크다운으로 글을 작성하면, 발행 가능한 Post가 된다

## 동작 원리

### 주요 개념

- **식별자**: Post의 고유 식별자. URL 경로로 사용
- **발행 정보**: 발행 상태와 식별 정보
- **본문**: 렌더링 가능한 형태의 글 내용

### 핵심 동작

- **조회**: 식별자로 개별 Post를 찾는다
- **목록화**: 발행된 Post들을 발행일 역순으로 나열한다
- **필터링**: 초안 상태의 Post는 독자에게 노출하지 않는다

## 경계

현재 명시적으로 제외한 것은 없다.

다음은 현재 스코프에 포함되지 않았으나, 미래 확장 가능한 영역이다:
- 시리즈/연재
- 카테고리 (현재는 태그만 존재)
- 작성자 정보
- 댓글/반응

## 하위 컨셉

### 발행 정보

글의 발행 상태와 식별 정보를 담는다.

- **식별자**: 고유 식별자
- **제목**: 글의 제목
- **태그**: 태그 배열
- **초안 여부**: 발행 전 상태인지 여부
- **발행일**: 발행된 날짜 (초안일 때는 없음)

### 본문

렌더링 가능한 형태의 글 내용이다.

- 원본은 마크다운으로 작성된다
- 독자에게는 HTML로 변환된 형태로 제공된다
- 원본 마크다운은 독자에게 필수적이지 않다
