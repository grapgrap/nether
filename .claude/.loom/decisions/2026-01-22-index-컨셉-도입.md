# 2026-01-22-index-컨셉-도입

## 맥락

loom 커맨드들이 작업을 시작할 때 프로젝트의 컨셉 구조를 파악할 수 있는 진입점이 필요했다.
현재는 `concepts/` 디렉토리의 개별 파일들만 존재하여, "무엇이 있는지" 한눈에 파악하기 어려웠다.

## 결정 내용

scope마다 `index.md` 파일을 두어 컨셉 목록과 관계 구조를 제공한다.

### 제약 조건

- loom 커맨드가 매번 디렉토리를 스캔하는 것은 비효율적
- 컨셉 간 관계는 대화 과정에서 점진적으로 드러남

### 전제 조건

- index는 "지도" 역할만 하고, 시작점 판단은 커맨드가 맥락을 보고 수행
- 새 컨셉 도출 또는 관계 변경 시 index도 함께 업데이트됨

## 대안

### 자동 생성 방식

- 설명: concepts/ 디렉토리를 스캔하여 매번 index를 재구성
- 트레이드오프: 관계 구조는 파일 내용을 파싱해야 알 수 있어 복잡도 증가. 또한 "핵심 구조"를 명시적으로 표현하기 어려움

### 히스토리 포함 방식

- 설명: index에 최근 변경 이력도 포함
- 트레이드오프: decision이 이미 변경 맥락을 추적하고 있어 중복. index의 역할을 단순하게 유지하기로 함
