# Design

컨셉의 구현 방법(How)을 정의한다.

## 목적

[[concept]]이 정의한 What/Why를 바탕으로, **어떻게 구현할 것인가**를 설계한다.
상세 코드 수준이 아닌, 모델링과 흐름 수준의 설계에 집중한다.

### 핵심 시나리오

컨셉이 정의되면, 해당 컨셉의 design을 작성한다.
design을 통해 구현에 필요한 구조와 흐름이 명확해진다.
[[plan]]의 역방향 흐름에서는 작업 분해 과정에서 설계 판단이 이뤄지며, 그 부산물로 design이 생성될 수 있다.

## 동작 원리

<!-- How(구현 방법)가 아닌 What(무엇을 달성하는가)에 집중한다. -->

### 주요 개념

- **모델링**: 상태, 엔티티, 관계 정의
- **데이터 흐름**: 입력에서 출력까지의 경로
- **비즈니스 로직**: 규칙, 조건, 분기

### 핵심 동작

- **설계**: 컨셉의 How를 구체화
- **협력 정의**: 하위 컨셉 간 상호작용 기술
- **결정 기록**: 설계 선택을 decision으로 남김

## 경계

- 상세 코드 구현 (구현 단계에서)
- What/Why 정의 ([[concept]]에서)
- 기술 스택 선택 (프로젝트 수준에서)

없음

## 관련 컨셉

- [[concept]] - design의 대상, 1:1 대응
- [[decision]] - 설계 과정의 선택을 기록
- [[plan]] - 정방향에서는 design이 plan의 입력, 역방향에서는 plan이 design을 생성
