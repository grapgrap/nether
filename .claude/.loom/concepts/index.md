# Index

scope 내 컨셉들의 목록과 관계 구조를 제공하는 지도.

## 목적

loom 커맨드가 작업을 시작하기 전에 프로젝트의 컨셉 구조를 파악하고, 사용자 맥락에 맞는 진입점(어떤 문서를 먼저 볼지)을 판단할 수 있게 한다.

### 핵심 시나리오

- loom 커맨드 실행 시 index를 먼저 읽으면, 현재 프로젝트에 어떤 컨셉들이 있는지 파악할 수 있다.
- 사용자의 요구사항과 index의 구조를 함께 보면, 어떤 컨셉 문서를 먼저 읽어야 할지 판단할 수 있다.

## 동작 원리

<!-- How(구현 방법)가 아닌 What(무엇을 달성하는가)에 집중한다. -->

### 주요 개념

- **컨셉 목록**: scope 내에 존재하는 모든 컨셉
- **관계 구조**: 컨셉 간의 연결 (조합, 동기화, 의존)

### 핵심 동작

- **조회**: loom 커맨드가 index를 읽고 컨셉 구조 파악
- **등록**: 새 컨셉 도출 시 index에 추가
- **갱신**: 기존 컨셉 간 새 관계 설정 시 index 업데이트

## 경계

- 작업 히스토리 추적 (decision에서)
- 시작점 마킹 (판단은 커맨드가 맥락을 보고 수행)
- 컨셉의 상세 내용 (개별 컨셉 문서에서)
