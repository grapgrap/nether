# Scope

specify가 동작하는 범위를 정의한다.

## 목적

모노레포 환경에서 스펙 문서의 위치와 작업 맥락을 결정한다.
요구사항이 어떤 서비스에 해당하는지 파악하여 적절한 `.loom/` 디렉토리를 선택한다.

## 동작 원리

### 결정 방식

1. **암시적 추론**: 대화 맥락에서 자동 추론
   - 명시적 서비스명 ("pop 서비스", "meeting")
   - 도메인 키워드 ("워크플로", "화상회의", "인증")

2. **명시적 지정**: `/loom.specify {서비스명}`으로 직접 설정

### 저장 구조

각 scope는 독립적인 `.loom/` 디렉토리를 가진다:

- `services/{서비스명}/.loom/` - 서비스별 스펙
- `.claude/.loom/` - specify 시스템 자체의 스펙

### Scope 탐색 흐름

1. 요구사항에서 키워드 추출
2. `services/` 디렉토리에서 관련 서비스 탐색
3. 명확하면 해당 서비스 사용, 불명확하면 사용자 확인

## 경계

- scope 간 상속/공유 미지원
- 물리적 디렉토리 구조에 종속
- 서비스 간 참조는 상대 경로로 가능하나 독립적 관리
